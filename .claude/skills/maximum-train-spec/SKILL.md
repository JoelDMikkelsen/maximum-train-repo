# Maximum Train Specification Compliance

This skill enforces the non-negotiable design principles from SPEC.md. **Every implementation decision must pass through these gates.**

## What This Is

Maximum Train is an emotionally intelligent, imagination-validating numerical universe designed for a highly intelligent, musically gifted, emotionally sensitive child. It is NOT a conventional game. It is a responsive cognitive environment.

The system must feel: Safe, Respectful, Beautiful, Predictable, Magical, Responsive.

**Never frustrating. Never patronizing.**

## Non-Negotiable Gates

### Gate 1: Zero Punishment Architecture

Before implementing ANY feedback mechanism, verify:

**FORBIDDEN - Never implement these:**
- Red X or error indicators
- Error buzzers or negative sounds
- The word "Wrong" or synonyms
- Failure states or game-over screens
- Loss of progress

**REQUIRED for incorrect interactions:**
- Gentle deflection animation
- Soft neutral tone
- Continued motion (never stop)

**REQUIRED for correct interactions:**
- Harmonic musical tone
- Visual particle reward
- Progress advancement
- Calm, satisfying feedback

**Test:** Would a sensitive child feel safe after getting an answer wrong? If no, redesign.

### Gate 2: Intelligence-Respectful Design

Before implementing ANY visual element, verify:

**USE:**
- Clean geometric forms
- Cosmic scale metaphors
- Elegant animation
- Mathematical beauty
- Dark backgrounds with subtle glow

**AVOID:**
- Cartoon overload / baby aesthetics
- Visual chaos or clutter
- Loud flashing visuals

**Test:** Would this feel like a planetarium, not a children's TV show?

### Gate 3: Imagination Validation

"Maximum Train" is a REAL number in this universe. It exists beyond Graham's Number.

- It is NOT fictional inside this system
- It must be treated as a legitimate discoverable entity
- When it appears, reality itself appears to break

**Test:** Does the implementation treat Maximum Train with the same seriousness as Googol or Graham's Number?

### Gate 4: Emotional Regulation Support

If no interaction occurs:
- After 5 seconds: Ambient animation increases slightly
- After 10 seconds: Gentle musical tone plays
- After 20 seconds: A train carriage slowly passes to invite interaction

**Never pressure. Always invite gently.**

## Four Core States

1. **Brick Swat Puzzle** - Floating brick clusters, tap to find quantities
2. **Train Progress** - Track at top, carriages added per correct answer
3. **Boss Reveal** - Zoom-out showing scale of large numbers
4. **Maximum Train** - Climax where reality breaks and train extends infinitely

## Technical Requirements

- Target: iPad Safari, 60fps
- Tech: Plain HTML, CSS, JavaScript (no framework)
- Rendering: Canvas preferred for animations
- Animation: requestAnimationFrame
- Audio: Web Audio API
- Touch: touchstart events
- Avoid excessive DOM operations

## Boss Number Sequence

100 -> 1,000 -> 1,000,000 -> 1,000,000,000 -> Googol -> Graham's Number -> Maximum Train

(V1 simplified: 100 -> 1,000 -> Maximum Train)

## Extensibility

- New boss numbers must be easy to add
- Maximum Train is NOT hardcoded as absolute limit
- System must remain expandable for: TREE(3), Busy Beaver, child-invented numbers
